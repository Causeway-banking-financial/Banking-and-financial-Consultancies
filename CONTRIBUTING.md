# Contributing

Thank you for helping improve Causeway Banking Financial. This repository
focuses on standards and readiness documentation that guide production delivery.

## How to contribute

1. Open an issue describing the change or problem.
2. Create a short-lived feature branch.
3. Keep changes focused and documented.
4. Open a pull request and request review from code owners.

## Documentation changes

- Update related sections in `docs/` when changing standards.
- Record architectural decisions in ADRs when needed.
- Ensure changes do not weaken security or compliance expectations.

## Security

If you discover a vulnerability or security concern, do not open a public issue.
Follow the guidance in `SECURITY.md`.
